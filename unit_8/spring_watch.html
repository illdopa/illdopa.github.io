Spring For Apple Watch

Concept, User Testing, Wireframes, Prototypes, UI Design

Spring: A rhythm-based music service for exercise (iOS)


Problem

  • Tiny and novel interface
  • Technical limitations different from iPhone

Goal

Create a simple and clean watch experience that provides utility to the user in ways unavailable to the standard iPhone app.

Solution

  • Allow user to set up, perform and complete an activity entirely iPhone free.
  • Display only the most useful live-time GPS and music data.
  • Facilitate easy track skipping from the watch face mid-activity.
  • Streamline user experience post watch release

Strategy

Given the cumbersome nature of interacting with the iPhone during exercise, we sought out to allow Watch users to conduct a full workout cycle entirely from the watch. A full workout cycle is comprised of 3 components:

  1. Setup
  2. Workout
  3. Workout Summary

Low-Level Wireframe 1

included many features from the phone app while also including design touches native to the phone app.

Low-Level Wireframe 2

Early wireframes were printed out on paper to exact scale of the watch face (Apple Watch was not yet available.)

In order to de-clutter interface and decrease cognitive load while on the move, the least essential features were nixed and a swipe-able play view was introduced.

Play view can be customized to display music info,workout data or GPS data.

Medium Level Wireframe (Setup)

One concept borrows fastlane UI components from the phone app.

Another concept explores toggles, which utilizes single taps and a simplified UI that doesn’t require advanced scrolling/animation.

Music customization has been removed to simplify experience.

Setup (Iteration 1)

Usability of setup flow was improved by borrowing the scrolling cell design used by other native Apple Watch apps.

Default or user saved BPM values are stored for each activity.

Setup (Iteration 2)

Cell color changed to blue to increase visibility during outdoor activites.

Start workout button is enlarged to increase ‘tappability.’

Play View (Iteration 1)

Further detail on customizable Play View. (3 page modal sheet)

Play View (Iteration 2)

Buttons were enlarged slightly and given orange background to increase visibility during outdoor activities.

Play View analytics and usability testing for iPhone users showed that track skip was the most popular interaction, so a transparent, full screen skip button was overlaid onto music view.

GPS view was nixed due to technical and usability reasons.

Post Workout View

Static reward-trophy is replaced with a short sprite kit animation, which fades directly into core exercise data.

Done button takes user back to Setup view.
